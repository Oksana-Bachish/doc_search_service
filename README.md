## Документ-Поисковик (Микросервис полнотекстового поиска)

Асинхронный микросервис на FastAPI, реализующий полнотекстовый поиск по архивам документов с использованием ElasticSearch и PostgreSQL. 

Проект спроектирован по микросервисной архитектуре, полностью контейнеризирован и готов к деплою на удаленный сервер.

### 🛠 Стек технологий
- **Backend:** Python 3.10 / FastAPI (асинхронные роуты)
- **Database:** PostgreSQL 17 (на базе Linux Debian Bookworm)
- **Search Engine:** ElasticSearch 9.4.1
- **DevOps:** Docker / Docker Compose / Изолированные внутренние сети
- **Security:** Безопасное управление конфигурацией через `.env` переменные
- **Testing:** Автотесты на базе фреймворка pytest

### 📐 Архитектура и логика работы
1. **Хранение данных:** PostgreSQL используется как отказоустойчивое реляционное хранилище для всех полей документов (`id`, `rubrics`, `text`, `created_date`).
2. **Поисковый индекс:** ElasticSearch используется строго для полнотекстового поиска по полю `text`. 
3. **Логика поиска:** При входящем текстовом запросе FastAPI асинхронно обращается к ElasticSearch, получает массив релевантных `id`, после чего выполняет точечную выборку документов из PostgreSQL, сортирует их по дате создания и возвращает клиенту валидный JSON.

### 🚀 Как запустить проект
1. Выберите папку, куда будете клонировать Git-репозиторий:
```bash
cd <имя_папки>
```
2. Клонируйте репозиторий:
```bash
git clone https://github.com/Oksana-Bachish/doc_search_service.git
```
3. Перейдите в директорию проекта:
```bash
cd doc_search_service
```
4. Создайте файл `.env`:
```bash
touch .env
```
5. Добавьте в `.env.dev` следующие переменные:
```env
POSTGRES_DB=your_db_name
POSTGRES_USER=your_db_user
POSTGRES_PASSWORD=your_db_password
```
6. Поместите ваш исходный массив данных (файл датасета в формате `posts.csv`) в корневую директорию проекта. Файл должен содержать текстовые документы со структурой полей: `rubrics`, `text`, `created_date`. 

7. Для развертывания всей инфраструктуры одной командой выполните в терминале:
```bash
docker compose up --build -d
```
8. Для наполнения базы данных и создания поисковых индексов из тестового массива данных (`posts.csv`) запустите изолированный ETL-скрипт **внутри уже запущенного Docker-контейнера**:
```bash
docker compose exec web python load_data.py
 ```

### 🧪 Запуск функциональных тестов (pytest)
В проекте реализованы асинхронные функциональные тесты на базе фреймворка `pytest` и клиента `httpx` (спецификация `ASGITransport`), проверяющие корректность работы поискового эндпоинта и логику обработки ошибок (404 Not Found) при удалении документов.
Для запуска тестов выполните команду:
```bash
docker compose exec web python -m pytest
```

### 📑 Документация API
После запуска интерактивная документация Swagger доступна по адресу: `http://127.0.0.1:8000/docs`.

### Основные эндпоинты:
- `GET /search?text=...` — Полнотекстовый поиск по документам. Возвращает **первые 20 наиболее релевантных документов** со всеми полями из БД, отсортированных по дате создания (`created_date DESC`).
- `DELETE /document/{doc_id}` — Удаление документа по его ID параллельно из PostgreSQL и поискового индекса ElasticSearch.


Полная спецификация OpenAPI сохранена в файле `docs.json`.
